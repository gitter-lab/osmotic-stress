Load the TimeXNet summary statistics file¶

TimeXNet paper suggests chooising the "sub-network with the highest number of source nodes and less than 1% unreliable edges (edge weight < 0.5)"

In the output file (per http://timexnet.hgc.jp/):

  • Sources: Total number of "initial" genes in the predicted response network
  • Edges: Total number of edges predicted in the response network
  • Edges0.5: Total number of edges with reliability less than 0.5 predicted in the response network

We used a different PPI network so these thresholds are not applicable exactly

Assess the fraction of unreliable edges at a 0.3 threshold¶

Still are not any networks with < 1% unreliable edges but there are 11 networks with < 5% unreliable edges. They are all of the networks with Gamma2 = 0. Gamma2 controls the flow through the intermediate proteins, so is this constraint is not being enforced in these subnetworks?

Select the optimal parameters¶

A single network has the min unreliability score and the maximum number of sources (for the networks with unreliability scores < 5%) so choose that network. From the edge distribution above, we see that it is one of the smallest networks. There must be a flow threhsold applied to these edges counts because the output file for this network shows 1073 edges.
